Every map record must be defined through the following data attributes in order to be displayed on the statistics section of the website. The following table gives you an understanding on how to format the data for each map record and what is required from each attribute. Maps must be ordered in alphabetical order within the data file.

Overcast bow records: src/data/stats/overcast/bow.json
Stratus bow records (unused): src/data/stats/stratus/bow.json
Avicus bow records (unused): src/data/stats/avicus/bow.json

Main record attributes
Attribute Description Value Default
"name": The name of the map which the record is for. String
"slug": The map's unique slug ID, which is generally auto-generated by the associated network and is used to reference the map in other placed on the website. String
"tags": [] The tags which describe the map's game mode and other features. Tag Data
"record_first": [] Details for the player holding the top record. Player Data
"record_second": [] Details for the player holding the second place record. Player Data
"record_third": [] Details for the player holding the third place record. Player Data
Players
Provides information about who holds the record and the distance in which they achieved that record with. Only the author name is required, however to reduce the amount of work required to keep names up-to-date, it is recommended to also provide the player UUID in addition to the player's display name.
Attribute Description Value Default
"username": The player's current display name. String
"uuid": The player's UUID, which can be found at mcuuid.net String
"distance": The distance in which the player holds the record for. String
"evidence": Additional evidence of the record which is displayed next to the record itself. URL
